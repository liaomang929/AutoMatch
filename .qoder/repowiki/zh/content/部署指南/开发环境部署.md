## 项目概述

AutoMatch是一个基于React + Vite + Express的足球赛事智能分析工具，专为足球竞彩分析师设计。该项目集成了赛事数据抓取、智能选场、AI辅助分析、文案生成等功能，帮助分析师高效完成每日赛事分析工作。

**项目特点：**
- 前端：React + Vite + Ant Design
- 后端：Node.js + Express
- 数据抓取：Puppeteer（无头浏览器）
- AI服务：智谱GLM-4大模型API
- 数据存储：本地文件系统（JSON/Markdown格式）

## 系统要求

### 硬件要求
- **处理器**：Intel Core i5 或同等AMD处理器

- **npm**：随Node.js自带
- **yarn**：可选，需单独安装
- **pnpm**：可选，需单独安装

- Firefox 88+

## 后端服务配置

### 1. 后端核心配置

后端服务基于Express框架，主要配置位于`server/index.js`：

```javascript
// 服务器端口配置
const PORT = process.env.PORT || 3001;

// CORS跨域配置
app.use(cors());

// JSON解析配置（支持10MB大小）
app.use(express.json({ limit: '10mb' }));

// 静态文件服务配置
const dataDir = process.env.DATA_DIR || path.join(require('os').homedir(), 'Desktop', 'AutoMatch');
app.use('/data', express.static(dataDir));
```

## 常见问题排查

### 1. 端口占用问题

**问题**：端口被占用导致服务无法启动

**解决方案**：
```bash
# 检查端口占用
lsof -i :3001
lsof -i :5173

# 杀死占用进程
kill -9 <PID>

# 或者修改端口
export PORT=3002
```

### 2. 依赖安装失败

**问题**：npm install失败

**解决方案**：
```bash
# 清理缓存
npm cache clean --force

# 删除node_modules重新安装
rm -rf node_modules
rm package-lock.json
npm install

# 或者使用yarn
yarn install
```

### 3. 跨域问题

**问题**：前端无法访问后端API

**解决方案**：
```javascript
// 检查Vite代理配置
// client/vite.config.js 中的代理设置

// 确保后端CORS配置正确
// server/index.js 中的CORS中间件
```

### 4. 数据目录权限问题

**问题**：无法写入数据文件

**解决方案**：
```bash
# 检查目录权限
ls -la ~/Desktop/

# 授予写权限
chmod 755 ~/Desktop/AutoMatch

# 或者修改数据目录
export DATA_DIR=/tmp/AutoMatch
```

### 5. Puppeteer依赖问题

**问题**：Puppeteer相关错误

**解决方案**：
```bash
# 安装系统依赖
# macOS: brew install chromium
# Ubuntu: sudo apt-get install chromium-browser
# CentOS: sudo yum install chromium

# 或者使用puppeteer-core
npm install puppeteer-core
```

### 6. 环境变量未生效

**问题**：环境变量设置无效

**解决方案**：
```bash
# 检查.env文件格式
cat .env

# 确保没有空格
PORT=3001

# 重启服务使配置生效
```
